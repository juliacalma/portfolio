Statistics for Data Science — Case Study (COMP616 / STATS604)

I used R (in RStudio) to answer a set of statistics questions using non-parametric methods. I kept code and explanation together in an R Markdown file and exported a clean PDF report.

Problem

Analyse small samples and categorical data without assuming normality. Answer questions about association, medians, group differences, and independence using rank-based and distribution-free tests (α = 0.05).

Data & Tools

Tools: R, RStudio, R Markdown.
Data: small numeric samples and contingency tables (multiple tasks).
Outputs: tidy code chunks, inline results, and a knitted PDF report.

Methods

  • Rank correlation: Spearman’s ρ and Kendall’s τ to test monotonic association.
  • Sign test: two-sided test for a median against a stated value.
  • Runs test: randomness/independence in a sequence.
  • Wilcoxon rank-sum: two-sample comparison (handles non-normal data and ties).
  • Kruskal–Wallis: multi-group comparison of distributions/medians.
  • Chi-squared independence: association between two categorical variables (checked expected counts; used appropriate corrections where needed).

My Role

  • Wrote all R code and set up each hypothesis test with clear H₀ / H₁ statements.
  • Validated assumptions (measurement scale, independence, expected counts) and handled ties.
  • Produced supporting visuals (boxplots, rank scatter, contingency summaries) and explained results.
  • Compiled a reproducible R Markdown report with code, outputs, and commentary.

Results (highlights)

  • Rank tests identified weak/no monotonic relationships where Pearson would be unsuitable.
  • Sign test flagged a median shift at α = 0.05 in one task; runs test supported randomness in another.
  • Wilcoxon showed no strong two-group difference in one comparison; Kruskal–Wallis detected at least one differing group across multiple groups.
  • Chi-squared found evidence of association in a 3×3 table; residuals pointed to which cells drove the effect.

Learnings

  • When to use non-parametrics: good fit for small n, skewed data, and ordinal scales; independence of observations still matters.
  • Don’t stop at p-values: add an effect size (e.g., rank-biserial for Wilcoxon, η² for Kruskal–Wallis, Cramér’s V for χ²).
  • After Kruskal–Wallis: use Dunn tests with Holm–Bonferroni to see which groups differ..
  • Ties & exact vs. asymptotic p-values: ties are common; where possible, use exact methods or continuity corrections and state which was used.
  • χ² checks: make sure expected counts aren’t too small; if they are, use Fisher’s exact.
  • Use clear visuals: box/violin plots and ECDFs for numbers; mosaic/association plots for χ².
  • Report robust summaries: median and IQR; add a bootstrap CI for the median if needed.
  • Power & sample size: non-parametric tests often need larger n for the same power; effect sizes guide planning.
  • State hypotheses upfront: write H₀/H₁ clearly, choose one- vs two-sided before testing, and keep α consistent.
  • Reproducibility: keep all code + narrative in R Markdown, set a seed, include sessionInfo(), and knit to a shareable PDF.
